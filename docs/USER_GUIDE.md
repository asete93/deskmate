# Deskmate 사용자 가이드

> **전제**: 서버에 **Claude Code가 설치·로그인**되어 있어야 하며, 보안상 **내부망/VPN 안에서 사용**하는 것을 권장합니다.
> 이하 "**데이터 폴더**"는 기본 `~/.claude-control/<이름>/`이고, `--data <경로>`로 지정하면 그 경로입니다. 실제 경로는 기동 배너와 설정 화면(비밀번호 분실 안내)에 표시됩니다.

## 1. 시작하기

```bash
npx github:asete93/deskmate \
  --port auto --name myproject \
  --allow 100.64.0.0/10,192.168.0.0/16,127.0.0.1/32 \
  --https --lang ko --driver sdk
```

- `--port auto` 남는 포트 자동 할당 / `--name` 데이터 공간 분리 / `--data` 데이터 경로 직접 지정
- `--allow` 접근 허용 CIDR — 차단된 요청은 서버 콘솔에 `접근 차단: <IP>`로 기록됩니다. 전체 허용은 생략(비권장)
- `--https` 자체 서명 TLS(클립보드 등 secure 기능) / `--lang ko|en` 시스템 언어 / `--no-terminal --no-files` 해당 기능 완전 비활성
- 인증(하나 선택): `claude /login` 된 머신이면 자동 감지 · `CLAUDE_CODE_OAUTH_TOKEN`(권장, `claude setup-token` 발급) · `ANTHROPIC_API_KEY`
- 인증 없으면 mock 드라이버(UI 미리보기)로 기동

## 2. 핵심 구조

| 역할 | 누구 | 하는 일 |
|---|---|---|
| **대표** | 당신 | 지시·결재·검토 |
| **팀장** | 메인 에이전트 | 분석·분해·브리프·검증·보고. **파일 편집이 서버 차원에서 차단** — 반드시 위임 |
| **팀원** | 서브 에이전트 | 브리프 받아 구현·검증, 팀장에게 보고 |

- **팀원 생성 경로는 3가지뿐**: ① 팀장 결재 상신 → 대표 승인(승인 화면에서 모델·effort 조정 가능) ② 조직도에서 대표 직접 고용 ③ 외부 AI(OpenAI Codex) 연동. 그 외(숨은 서브에이전트 등)는 서버가 거부.
- **실행 모드**(채팅 ⚙): Plan(계획 승인 카드 후 실행) / Auto(파일 수정 자동 수락) / Ask(단계별 확인).
- **지침 4계층**: 서버 로직 → 불변 플랫폼 지침 → 프로젝트 CLAUDE.md(채팅 화면 탭에서 편집) → 런타임 설정. 충돌 시 앞이 우선.

## 3. 채팅

- **방 = 독립 기억.** 방마다 팀장 세션이 분리됩니다. 방별 **MODEL·EFFORT 오버라이드**(⚙ 팝업 — 작업 중 변경 시 진행 턴은 중단됨). 다른 방의 결정은 팀장이 이력 검색 도구로 참조합니다.
- **@이름**으로 특정 팀원 지목, 입력창의 대상 pill로 수신자 고정.
- **긴 텍스트 붙여넣기**(8줄+ 또는 600자+)는 `📋 붙여넣은 텍스트 · N줄` 칩으로 접힘 — 클릭해 원문 확인, 에이전트에겐 원문 전체 전달.
- Shift+Enter 개행 · 파일/이미지 드래그·붙여넣기 첨부 · 발신자 아바타 클릭 = 그 팀원 설정 팝업.
- **카드**(선택/폼/파일 수정 승인/산출물 검토/계획 승인): 답을 고르는 대신 **새 메시지를 보내면 그 내용이 답변으로 주입**되어 흐름이 막히지 않습니다. 긴 본문은 요약+"전체 내용 보기" 팝업으로 표시.
- 헤더 버튼: **대화 내보내기**(현재 방 → Markdown 파일), **대화 초기화**(전체 초기화 = 내용+기억 / 내용만 지우기 = 기억·진행 작업 유지).

## 4. 조직도·팀원 관리

- 팀원 목록/조직도에서 카드 클릭(조직도는 "채팅 이동 / 모델·설정 변경" 선택 팝업) → **설정 팝업**: 이름 · 아바타(1~4자·이모지) · 역할 · 커스텀 지침 · 모델 · effort.
- **스펙 내보내기(JSON)**: 잘 다듬은 직원의 역할·지침·모델 스펙을 파일(`deskmate-member-이름.json`)로 저장해 공유.
- 해고: 팀원 카드의 해고 버튼(대표 권한, 즉시) 또는 팀장이 결재 상신.
- 팀원 1:1 대화 초기화: 세션이 비대해져 토큰이 낭비될 때 사용.

## 5. 업무 추적

- **티켓**: 팀장이 위임할 때마다 **자동 생성** → 팀원 응답 시 검토(review) → 요청 완료 시 done. 수동 등록·조정도 가능. 보드/테이블 뷰.
- **요청(REQ)**: 대표의 작업 요청 단위 — 대화·토큰·보고서가 묶입니다.
- **보고서**: REQ 완료 시 등록 — 웹 열람(**테마 4종**: 클래식/문서/대시보드/다크) + **PPTX/Excel 다운로드**.
- **결재함**: 고용/해고/방향 결정/기타. 결재 이력 화면 포함.

## 6. 산출물 핀 리뷰

웹 산출물 검토 카드에서 "리뷰"로 진입 —
- **우클릭 = 핀 코멘트**(좌클릭은 링크 등 원래 동작 유지), 핀은 대상 요소를 하이라이트하고 우측 목록에서 클릭하면 해당 위치로 스크롤.
- ✏️ 모드에서 텍스트 클릭 = **그 자리에서 직접 수정**.
- 제출하면 요소 셀렉터·수정 전/후가 담긴 **구조화된 지시서**가 팀에 전달되고, 반영본이 다시 검토로 돌아옵니다.
- 화면보다 넓은 시안은 자동 축소되어 전체가 보입니다.

## 7. 파일

워크스페이스 한정 탐색기 + CodeMirror 에디터(문법 하이라이트, ⌘S 저장).
트리 DnD 이동 · 외부 파일 드래그 업로드 · 다운로드 · 다중 선택(Ctrl/Shift 클릭, 빈 공간 드래그 범위) · **⌘C/⌘X/⌘V/Del** · 우클릭 메뉴 · 클립보드 파일 붙여넣기 업로드. 모바일은 에디터 중심 + 파일트리 팝업. **기본 off — 설정 → 메뉴 표시에서 켜세요.**

## 8. 터미널

서버 셸 웹 접속. 가로/세로 분할 · 창별 폰트(Ctrl+휠) · 헤더 DnD 배치 · 복사/붙여넣기(HTTPS 필요) · 휠 스크롤백 · 새창 팝업(창 닫으면 종료). X는 화면 창만 닫고 서버 세션은 유지되며, 30분간 재접속 없으면 자동 정리. **기본 off.**

## 9. Git

- **커밋 이력**: 브랜치 · 커밋 그래프 · 커밋별 diff · 파일 트리(접기/펼치기, 커밋 시점 스냅샷).
- **변경사항 · 커밋**: 워킹트리 파일별 스테이지/내리기 · diff 미리보기 · `.gitignore` 편집(저장 즉시 반영) · 커밋. **커밋 메시지를 비워두면 변경 내용을 분석해 자동 생성**(Haiku 1회 — 소량 토큰, 실패 시 규칙 기반 폴백은 토큰 0).
- **기본 off.** 서버에 git이 없으면 메뉴가 자동 비활성됩니다(설정에 안내 표시).

## 10. 설정

- **서비스 연결**: 다른 포트/서버의 인스턴스를 등록해 사이드바 상단 스위처로 전환.
- **언어**: 한국어/English — UI, 에이전트 응답 언어, 시드 기본 명칭(팀장↔TeamLead, 메인 채팅↔Main Chat — 기본값 그대로일 때만)까지 전환됩니다.
- **로그인**: 단일 비밀번호(scrypt 해시), 5회 실패 = 15분 잠금. 분실 시 서버에서 `touch <데이터폴더>/reset-password` 후 다음 로그인 시도 때 재설정.
- **예약 작업**: 1회/매일/매주 지정 시각에 팀장(또는 팀원)에게 자동 지시.
- **메뉴 표시**: Git/터미널/파일 토글 — 끄면 메뉴만이 아니라 API까지 차단. (기동 옵션 `--no-*`로 끈 기능은 여기에도 표시되지 않음)
- **위험 구역**: **전체 기억 초기화**(대화·티켓·파일 유지, 에이전트 세션 기억만 리셋 — CLI 자동 메모리 포함) / **전체 데이터 초기화**(완전 삭제, 확인 단어 입력 필요).

## 11. 사용량·토큰

- 우하단 위젯(모바일: 하단 탭 "사용량")에서 구독 한도(세션/주간) · 리셋 시각 · 오늘 토큰 확인. REQ별 토큰 집계.
- 절약 팁: 잡담·간단한 방은 방 스펙을 Haiku/low로 / 반복 작업 팀원은 저사양 모델로 고용 / 세션이 길어지면 기억 초기화 / 리뷰류는 Codex 팀원으로 오프로드.

## 12. 배포

상시 운영은 systemd 권장:

```ini
# /etc/systemd/system/deskmate.service
[Unit]
Description=Deskmate
After=network.target
[Service]
Environment=CLAUDE_CODE_OAUTH_TOKEN=<토큰>
ExecStart=/usr/bin/npx github:asete93/deskmate --port 3200 --name prod --https --allow <신뢰대역>
Restart=always
User=<전용계정>   # root 대신 권한 최소화 계정 권장
[Install]
WantedBy=multi-user.target
```

- 업데이트: `rm -rf ~/.npm/_npx` 후 재기동(또는 `npm i -g github:asete93/deskmate` 재실행). 데이터는 데이터 폴더에 있어 안전.
- 완전 초기화: 서비스 중지 → 데이터 폴더 삭제 → 재기동.
- 보안 하드닝: 내부망/VPN 우선, 공개 시 로그인+`--allow`+TLS 리버스 프록시, systemd `ProtectSystem=strict`·`NoNewPrivileges=yes` 권장.

## 13. 문제 해결

| 증상 | 조치 |
|---|---|
| 접속하면 forbidden | 접속 IP가 `--allow` 밖 — 서버 콘솔의 `접근 차단: <IP>` 로그를 보고 그 대역을 추가. 전체 허용은 `--allow` 생략 또는 `0.0.0.0/0` |
| Node 버전 오류로 종료 | Node 22.5+ 필요 — `nvm install 22 && nvm use 22` |
| Git 메뉴가 없음 | 서버에 git 미설치(설정 화면에 경고) — 설치 후 재시작 |
| Codex 팀원이 응답 없음 | `npm i -g @openai/codex` + `codex login` |
| 에이전트가 엉뚱한 과거 맥락 언급 | 위험 구역 → 전체 기억 초기화 |
| 새 커밋이 npx에 반영 안 됨 | `rm -rf ~/.npm/_npx` 후 재실행 |
| 비밀번호 분실 | `touch <데이터폴더>/reset-password` → 다음 로그인 시도 때 재설정 |
